# 阶段 2.5 实验总结

更新日期：2026-03-31

## 1. 目的

阶段 2.5 的目标不是直接证明 `Cycle` 一定提升总体回归精度，而是验证：

1. 交替更新是否能修复阶段 2 中同步联合训练的干扰问题。
2. 是否存在一个足够稳定的联合训练主干，可作为阶段 3 接入 `L_hill` 的母体配置。

因此，本阶段采用漏斗式推进：

- `2.5A`：激进版
- `2.5B`：平衡版
- `2.5C`：保守版

所有实验均使用与 `Model C` 相同的 split 和同一组 seed：

- `20260325`
- `20260331`
- `20260407`

## 3. 阶段 2.5 正式结果

### 3.1 2.5A：激进版

三 seed 汇总：

- `MAE = 6.5043 ± 0.2778`
- `RMSE = 13.7021 ± 0.5030`
- `R2 = 0.7481 ± 0.0187`

最佳单次：

- 最佳 seed：`20260407`
- `MAE = 6.1848`
- `RMSE = 13.3872`
- `R2 = 0.7597`

结论：

- `2.5A` 没有形成明确总体收益。
- 平均 `RMSE` 相对 `Model C` 劣化约 `3.38%`。
- 平均 `R2` 相对 `Model C` 下降约 `0.0114`。
- 不满足路径 A，也不满足路径 B。

### 3.2 2.5B：平衡版

三 seed 汇总：

- `MAE = 6.3997 ± 0.0728`
- `RMSE = 13.5490 ± 0.1472`
- `R2 = 0.7539 ± 0.0054`

最佳单次：

- 最佳 seed：`20260325`
- `MAE = 6.3388`
- `RMSE = 13.4417`
- `R2 = 0.7578`

结论：

- `2.5B` 是 `2.5A/2.5B` 中最稳定的配置。
- 平均 `MAE` 相对 `Model C` 改善约 `1.25%`。
- 但平均 `RMSE` 仍劣化约 `2.23%`，略高于路径 B 的 `2%` 上限。
- 平均 `R2` 相对 `Model C` 下降约 `0.0056`，在容许范围内。
- 因此，`2.5B` 非常接近路径 B，但严格按既定门槛仍未通过。

### 3.3 2.5C：保守版

三 seed 汇总：

- `MAE = 6.5492 ± 0.0000`
- `RMSE = 13.4009 ± 0.0000`
- `R2 = 0.7592 ± 0.0000`

最佳单次：

- 最佳 seed：`20260407`
- `MAE = 6.5492`
- `RMSE = 13.4009`
- `R2 = 0.7592`

结论：

- `2.5C` 的 Predictor 完全冻结，因此结果在三 seed 下完全一致。
- 相对 `Model C`：
  - `MAE` 劣化约 `1.06%`
  - `RMSE` 劣化约 `1.11%`
  - `R2` 下降约 `0.0003`
- 在保守路径下，`2.5C` 满足“总体不明显退化且训练成功率 100%”的要求。
- 因此，`2.5C` 是当前最适合作为阶段 3 母体配置的版本。

## 4. A/B 优化尝试结论

2026-03-31 对 `2.5B` 又进行了小范围继续优化，重点尝试了：

- 调整 `generator_lr`
- 调整 `w_cycle`
- 调整 `w_recon`
- 调整 `predictor_lr`
- 调整 `w_mono`
- 增加 `p_steps`
- 增加 `joint_epochs`

结论如下：

1. 在当前 alternating 实现下，Generator 侧超参数对最终 test 回归指标影响很弱。
2. Predictor 侧超参数改动能改变结果，但没有把 `RMSE` 压到优于 `2.5B` 基线。
3. `2.5B` 原始基线仍是 `A/B` 范围内的最优可用配置。
4. 继续投入 `A/B` 的边际收益已经很低，不建议再围绕阶段 2.5 做小步调参。

## 5. 阶段结论

阶段 2.5 的正式结论建议写法如下：

1. 交替更新确实改善了训练稳定性叙事，尤其在 `2.5B` 和 `2.5C` 中，比 `2.5A` 更可控。
2. 但当前没有证据表明交替更新能够稳定地在总体回归指标上超越 `Model C`。
3. `2.5B` 接近“可接受放行”，但严格按预先定义门槛，仍未完全通过。
4. `2.5C` 虽然没有带来精度收益，但提供了一个干净、稳定、可复用的联合训练主干。
5. 因此，若继续推进阶段 3，建议采用 `2.5C` 作为接入 `L_hill` 的母体配置。
